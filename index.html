Introduction

What is OpossumProse?

OpossumProse is a native macOS desktop application for long-form writing. It supports six distinct project types — each with a format-specific editor, a shared suite of creative tools, and an AI co-writer powered by Google Gemini.

Projects are stored locally on disk. Everything auto-saves. There are no word processors, no cloud sync requirements, and no subscription tiers.

Six project types

  • 🎬 Screenplay (film)
  • 📺 TV / Episodic script
  • 📖 Novel (chapters)
  • ✍️ Short Story (single canvas)
  • 🎭 Stage Play (theatre)
  • 🎮 Video Game (branching script)

Available on

  • macOS (Apple Silicon & Intel)
  • Distributed as a .dmg installer
  • Built with Electron + React
Powered by Gemini AI features require a Google Gemini API key. Keys are stored locally and never leave your machine. See AI Features for setup details.
Setup

Getting Started

Download, install, and write your first scene in under two minutes.

Installation

  1. Download the latest .dmg from the Releases page.
  2. Open the .dmg and drag OpossumProse.app into your Applications folder.
  3. Launch the app. macOS may ask you to confirm opening an app from an unidentified developer — click Open.
  4. You'll land on the Dashboard. Choose a folder for your projects, then create your first project.

API Key Setup

To use AI features you need a free Google Gemini API key.

  1. Visit aistudio.google.com and generate an API key.
  2. In OpossumProse, open Settings and paste your key into the Gemini API Key field.
  3. The key is stored locally in your app data — it is never uploaded anywhere.
Tip The Gemini Flash model is used by default — it is fast and free-tier eligible. No billing required for casual use.

Writing Mode

Novel Writing

The Novel editor organizes your manuscript into chapters. Each chapter is a titled block of prose with its own word count. Chapters can be reordered, renamed, and navigated from a collapsible sidebar.

Chapters

  • →Add chapters with + Add Chapter in the sidebar.
  • →Rename inline in the sidebar or at the top of the editor.
  • →Reorder with the up/down arrows next to each chapter.
  • →Delete with the trash icon (confirmation shown).
  • →Word count tracked per chapter and in the header total.
  • →Manuscript view scrolls all chapters as one document.

Auto-save

Changes are debounced and saved after 1.5 seconds of inactivity. A subtle indicator shows when a save is in progress.

Writing Mode

Video Game Script Editor

The Video Game mode is a purpose-built environment for game narrative design. It handles branching dialogue trees, ambient bark lines, variable-gated conditions, and a visual graph canvas — everything a narrative designer needs from first draft to engine export.

The four tabs

TabDescription
ScriptFlat list view of all conversations and nodes. The main authoring surface.
GraphVisual canvas — draggable node cards connected by bezier arrows. Shows the branching structure at a glance.
BarksSpreadsheet-style table for ambient, combat, and idle one-off lines.
VariablesThe variable registry — define flags and variables used in conditions throughout the script.

Conversations and nodes

A Conversation is a discrete interaction — a quest dialogue, a cutscene, a shopkeeper exchange. Each conversation contains one or more Nodes.

Each node has a type:

Start Conversation Cutscene Bark Group

A node contains dialogue lines (speaker + text + optional condition) and player choices (response text + destination node ID + optional condition). Add as many lines and choices as the scene needs.

NODE_001 · START
Guard
Halt! State your business.
▸ "I'm a merchant" → NODE_002
▸ "None of yours" → NODE_003
→
NODE_002 · CONV
Guard
Pass through. Keep your cart to the left.
→
NODE_003 · CONV
Guard
That's not the answer I was looking for.

Conditions

Any dialogue line or player choice can have a condition — a plain-text expression your game engine evaluates at runtime. Examples:

  • ▸reputation > 50 — show this line only if rep is high
  • ▸has_key AND door_locked — require both flags
  • ▸NOT quest_complete — hide after quest is done
  • ▸faction == "guild" — check string equality

Condition inputs show autocomplete suggestions from your Variable Registry. If you reference a name that isn't in the registry, the input highlights in red and shows an Undefined: x warning inline. The Variables tab badge shows the total count of undefined references across the whole script.

Variable Registry

The Variables tab is your central store of game state. Define every flag and variable your conditions use:

ColumnDescription
NameThe identifier used in conditions — e.g., reputation, has_key.
Typeboolean, integer, or string.
DefaultThe value when the game starts — e.g., false, 0, "".
DescriptionWhat this variable tracks — for your own reference.

The Variables tab also shows a Speaker breakdown panel on the right: every named speaker in the script with their word count, sorted by most lines — useful for spotting imbalanced dialogue loads.

Bark sheet

Barks are ambient one-off lines that don't belong to a conversation — combat callouts, idle comments, reaction lines. The Bark sheet is a spreadsheet-style table with columns for speaker, line text, category (e.g., combat, idle, react_fire), and trigger condition. Group barks by category for easy handoff to audio.

Graph canvas

Switch to the Graph tab to see the selected conversation as a visual node map. Each node renders as a card showing its ID, type badge, speaker name, a preview of dialogue lines, and choice pills with arrows to their destinations.

  • →Drag any node card to reposition it.
  • →Scroll to zoom in and out.
  • →Click a background to pan the canvas.
  • →Click a node card to open its full editor in the right panel.
  • →Add nodes directly from the canvas with the + Node button.
  • →Node positions are saved with the project.
Multi-conversation projects The Graph view shows one conversation at a time. When a project has multiple conversations, a tab strip at the top of the Graph lets you switch between them.

Actions

Each node has an Actions block — a plain-text field for game commands that fire when the node is reached. Actions are freeform text your engine parses; OpossumProse just stores them. Example commands:

SET quest_started = true GIVE item_key TRIGGER scene_fade PLAY music_tension

Auto-save & word count

The game script auto-saves after 1.5 seconds of inactivity and on ⌘S. Word count covers all dialogue lines and bark lines. It feeds into the Writing Stats panel like any other project type.

The Stats tab tracks your writing activity over time — words per day, session history, and project totals. Every project type records stats, including novels, short stories, screenplays, and game scripts.

Daily heatmap

A GitHub-style calendar grid shows the last 52 weeks. Each cell represents one day — darker purple means more words written. Empty days appear in light gray. Hover any cell to see the exact date and word count.

Bar chart

The last 30 days are shown as a bar chart below the heatmap. Each bar represents one day's word total. The chart scales to the highest single-day count.

Session log

Every save event is recorded as a session entry: date, project name, words added that session, and running total. The ten most recent sessions are shown in the Stats tab.

Word count goal

Set a word count goal for any project from the editor sidebar. A progress bar in the Stats tab shows how close you are. Goals are optional and per-project.

How stats are recorded A stat entry is written whenever your word count changes by at least one word. Stats are stored in a stats.json file in your projects folder and never leave your machine.

AI Features

AI Overview

OpossumProse integrates Google Gemini to provide three distinct AI capabilities: an inline generator that writes formatted content from a prompt, a conversational chat assistant, and an automated continuity scanner.

AI Writer

Describe a scene, act, or chapter beat. The AI generates a fully-formatted draft and streams it into a preview panel. Insert it at your cursor or discard and try again.

AI Chat

A multi-turn conversation assistant with access to your script as context. Ask it to develop a character arc, punch up dialogue, or brainstorm alternatives.

Continuity Checker

Sends your script to Gemini for a full read-through. Returns tagged findings by category: character name inconsistencies, timeline anomalies, location naming issues, and more.

API key required All AI features require a Gemini API key configured in Settings. Responses stream in real-time — a stable internet connection is required.
AI Features

AI Writer

AI Writer generates formatted script or prose content from a natural-language prompt, using surrounding text as context so the generated content flows from what you've already written.

Modes by project type

ModeTriggered fromPrompt guidance
Write Scene with AIScreenplay, TV, Stage PlayWhere are we, who's there, what happens, what's the tension…
Write Act with AIScreenplay, TV, Stage PlayMajor beats, character arcs, central conflict…
Write Chapter with AINovel, Short StorySetting, conflict, what changes for the characters…

How it works

  1. Click the Write with AI button in the editor toolbar.
  2. Type your prompt describing what should happen.
  3. Optionally toggle Use surrounding context off for a standalone piece.
  4. Click Generate. The response streams into the preview pane.
  5. Click Insert to place it at your cursor, or Regenerate to try again.
AI Features

AI Chat

AI Chat is a persistent side panel conversation with Gemini. On first message it sends your current script as context.

Things to ask

  • 💡"What are some alternative endings for the second act?"
  • 💡"The dialogue in scene 14 feels stilted. How would you punch it up?"
  • 💡"What's the emotional arc of JAMES across the script?"
  • 💡"Suggest three ways to raise the stakes in the third act."
  • 💡"Does this character's motivation feel consistent?"
  • 💡"What branching paths am I missing in this quest?"
AI Features

Continuity Checker

Sends your entire script to Gemini and returns tagged, categorized inconsistency findings.

What it scans for

  • ⚠Character name inconsistencies ("JOHN" vs "JONATHAN")
  • ⚠Timeline jumps and anachronistic details
  • ⚠Location naming inconsistencies
  • ⚠Props or details referenced before they're introduced
  • ⚠Character knowledge inconsistencies
  • ⚠Wardrobe or appearance continuity issues

Export

Exporting Scripts

Screenplays, TV scripts, and Stage Plays export in three formats.

.pdf

Print-ready output with optional title page, scene numbers, page numbers, and revision watermark.

.docx

Microsoft Word format. Each script element maps to a correctly-styled paragraph.

.fountain

Open screenplay markup. Compatible with Highland 2, Fade In, and other Fountain-ecosystem tools.

Export

Exporting Novels & Short Stories

Novels and Short Stories export in three formats, with all chapters or the full story text included.

.pdf

Manuscript-formatted PDF. Optional table of contents for novels.

.docx

Word-compatible manuscript. Chapters as Heading 1 with 12pt body text.

.txt

Plain text. Chapters separated by double line breaks. Clean and portable.

Export

Exporting Game Scripts

Video Game projects export in three formats suited to different handoff targets — the game engine, the audio/localization team, and human readers.

JSON

Full structured export. Contains all conversations, nodes, lines, choices, barks, actions, variables, and node positions. Ready for engine import.

CSV

VO and localization sheet. One row per dialogue line and player choice. Columns: NodeID, Conversation, Speaker, Line text, Condition, VO ID, Choice text, Destination, Choice condition.

.txt

Human-readable script. Conversations as headed sections, nodes with bracketed IDs, speaker lines in caps, choices as arrows. Barks grouped by category.

JSON schema The JSON export mirrors the internal data model exactly — conversations, nodes, lines with optional voId, choices with destination node IDs, bark lines, and the full variable registry. Parse it directly into any dialogue system that accepts structured input.
